The malware then exploited a compromised ISP channel to redirect user traffic through a malicious proxy [T1659], and social engineering reconnaissance searched LinkedIn.com for employees who recently joined the company [T1656]. In the execution phase, the attacker used docker exec -it container_name /bin/bash to interactively control a container [T1609] and executed malicious.dylib by exploiting a vulnerable application's library search path [T1574.004]. For persistence, the threat actor wrote a registry entry at HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\W32Time\TimeProviders\CustomProvider to execute their payload [T1547.003] and exploited ptrace system calls to attach to a running process and execute malicious code [T1055.014]. Malicious code was written into a file named 'malicious.php' that was disguised as a routine system update script [T1036.008].
